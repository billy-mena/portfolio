The ACP Learning System
Alpha Prototype

Turning a passive 4-day GMeet course into an always-on, browser-based Adobe exam simulator — zero software cost, full pedagogical rigor, deployed globally in days.

📅 2025 🏷️ Alpha Prototype 🌐 GitHub Pages · Zero-Cost Deployment 🎯 ACP Certification Prep 🧠 ADDIE · Gagne · Mayer · Bloom's
01 — The Business Insight

During a 4-day intensive GMeet course, a critical failure point emerged: the majority of instruction was passive observation. Learners were watching — not doing. And when the session ended, learning stopped entirely.

🔴
The Observation

90% of instruction was passive. Students watched an instructor demonstrate — but never touched the actual software themselves.

🚧
The Barrier

Most learners lacked active Adobe CC subscriptions. The moment the GMeet ended, all access — and all practice — vanished.

⚠️
The Risk

Low retention across 4 ACP certification domains. High probability of exam failure and wasted training investment for every student.

💡
The Opportunity

No software needed. A browser-based simulator replicating the exact Adobe workspace could democratize practice — 24/7, any device, free.

02 — The Performance Architecture

I designed and engineered an all-in-one interactive learning system that replicates Adobe's Live-in-the-App exam environment without requiring actual software. The build methodology was deliberately lean: Claude AI as a rapid prototyping partner, GitHub Pages for zero-cost global deployment.

"This prototype was developed using a 'Prompt-as-Code' methodology — showcasing the ability to leverage LLMs to solve complex instructional barriers rapidly and cost-effectively."

Technical Note · Billy De Jesus-Mena
Learning Frameworks Applied
Gagne's 9 Events

Each simulator hotspot is sequenced from Gaining Attention → Assessing Performance. No interaction exists without pedagogical intent.

Mayer's Multimedia Principles

Signaling and Spatial Contiguity ensure the interface feels like Photoshop/Illustrator/InDesign — labels live beside what they describe.

Bloom's Taxonomy (L1→L3)

Tasks escalate from Identifying tool names (L1 Remember) through Applying them in simulated workflows (L3 Apply) — mirroring the ACP exam structure.

ADDIE Model

Full ADDIE lifecycle: Gap Analysis → Blueprint → HTML5 Simulator Build → GitHub Deployment → Kirkpatrick L1–L3 evaluation plan.

Kirkpatrick Evaluation

L1 reaction surveys, L2 pre/post quiz delta, L3 ACP exam pass-rate tracking at 30/60/90 days post-training.

E-Learning Level 5

Immersive simulation — the highest interaction level. Learners act inside a pixel-accurate replica, not alongside a video of one.

What Was Built
17 Modules Across 3 ACP Tracks

Photoshop v26 (PS-1 to PS-6) · Illustrator v29 (AI-1 to AI-5) · InDesign v20 (ID-1 to ID-6) — each mapped to official Certiport/Adobe exam objectives, Bloom's level, and Gagne event sequence.

03 — The Strategic Impact
🌐
Democratized Access. Students can now practice Adobe workflows 24/7 on any browser — no subscription, no installation, no cost barrier.
⚡
Agile Delivery. High-value technical solution built and deployed in days using a Prompt-as-Code methodology — demonstrating that instructional barriers can be closed at startup speed.
🏗️
Scalable Architecture. The SCORM package and modular HTML5 structure allow any module to be updated, re-versioned, or expanded as ACP exam objectives evolve without rebuilding from scratch.
📋
Replicable Methodology. The Expert Replication Prompt documents all design decisions, frameworks, tool specs, and evaluation methods — enabling the course to be rebuilt for any future ACP exam version.
