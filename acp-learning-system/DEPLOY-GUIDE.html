How to deploy this project

A complete step-by-step guide for first-timers. Three methods: GitHub Pages (free, recommended), local file use, and LMS/SCORM upload.

What you have

A self-contained folder of HTML files. No server is needed to use these — they work directly in a web browser. Here is the folder structure:

acp-2025-learning-system/ ├── index.html ← The home page. Always start here. ├── README.md ├── EXPERT-PROMPT.md ├── assets/ ← Shared CSS and JavaScript (DO NOT MOVE) │ ├── css/ │ │ └── global.css │ └── js/ │ ├── engine.js │ ├── progress.js │ └── scoring.js ├── photoshop/ ← Photoshop course │ ├── index.html │ ├── simulator/ d2–d6-*.html │ ├── slides/ ps-simulator-slides.html │ └── files/ task-briefs.html ├── illustrator/ ← Illustrator course ├── indesign/ ← InDesign course ├── shared/ ← Design theory module └── evaluation/ ← Kirkpatrick L1–L3 toolkits
⚠ Critical: Keep the folder structure exactly as-is. Every file uses relative paths to load the CSS and JavaScript from assets/. Moving files into different folders will break the styling and interactivity.

Method 1 — Use it locally (right now, no setup)

Easiest · Works offline · No account needed Open directly in a browser
  1. 1
    Download and unzip the project folder to your computer.
    Put it somewhere easy to find — your Desktop or Documents folder is fine.
  2. 2
    Open the folder. Find index.html in the root.
    It will be right at the top level — not inside any subfolder.
  3. 3
    Double-click index.html. It opens in your default browser.
    Chrome, Edge, Firefox, or Safari all work. If it opens in a text editor, right-click → Open with → your browser.
  4. 4
    You'll see the home page with the three tool cards (Photoshop, Illustrator, InDesign). Click any card to begin.
    Your progress is saved automatically in the browser's local storage — it will still be there when you come back, as long as you use the same browser on the same computer.
Note: Local use works perfectly for individual learners. The only limitation is that progress doesn't sync between computers or between learners. For classroom/group use, GitHub Pages (Method 2) is recommended.

Method 2 — GitHub Pages (free, shareable URL, recommended for instructors)

GitHub Pages hosts static HTML files for free and gives you a public URL like https://yourname.github.io/acp-2025/ that you can share with any learner. No coding experience required.

Free · Public URL · 5–10 min setup Step-by-step: GitHub Pages deployment

Part A — Create a GitHub account (skip if you already have one)

  1. 1
    Go to github.com and click "Sign up".
    Choose a username (this will appear in your URL), enter your email and a password.
  2. 2
    Verify your email address. GitHub will send a confirmation email — click the link in it.

Part B — Create a new repository

  1. 1
    Once logged in, click the + icon in the top-right corner → "New repository".
  2. 2
    Name the repository (e.g. acp-2025-learning-system). This name appears in your URL.
    Use lowercase letters and hyphens — no spaces.
  3. 3
    Set visibility to Public. (GitHub Pages only works on public repositories with a free account.)
  4. 4
    Leave "Add a README file" unchecked. Click "Create repository".

Part C — Upload the project files

  1. 1
    On your new (empty) repository page, click "uploading an existing file" in the centre of the page.
    If you can't see that link, click the Add file dropdown → "Upload files".
  2. 2
    Open your project folder in File Explorer (Windows) or Finder (Mac). Select all files and folders inside the project folder and drag them into the GitHub upload area.
    Make sure you are dragging the contents of the folder, not the folder itself. The upload area should show: index.html, assets/, photoshop/, illustrator/, etc. at the top level.
  3. 3
    Wait for all files to upload (progress bars). Then scroll down and click "Commit changes".
    You can leave the commit message as the default "Add files via upload".

Part D — Enable GitHub Pages

  1. 1
    Click the Settings tab at the top of your repository.
  2. 2
    In the left sidebar, scroll down and click Pages.
  3. 3
    Under "Source", click the dropdown that says "None" and select "Deploy from a branch".
  4. 4
    In the branch dropdown, select "main". In the folder dropdown, select "/ (root)". Click Save.
  5. 5
    Wait 1–3 minutes. Refresh the Settings → Pages page. Your live URL will appear:
    https://YOUR-USERNAME.github.io/acp-2025-learning-system/
✓ That's it! Share that URL with your learners. The site is live and free. Any time you update a file in the repository (upload a new version), the live site updates within a few minutes automatically.

Updating files after initial upload

  1. 1
    Go to your repository on github.com. Navigate into the subfolder containing the file you want to update (e.g. click photoshop/ → files/).
  2. 2
    Click the file name. Then click the pencil icon (Edit) in the top-right of the file view, or use the "..." menu → "Replace file" to upload a new version.
  3. 3
    Click Commit changes. The site updates within 1–3 minutes.

Method 3 — Upload to an LMS (Moodle, Canvas, Blackboard)

If you want learners to access this through your institution's Learning Management System and track completion, use the SCORM packaging method.

Moodle · Canvas · Blackboard · Any SCORM LMS SCORM packaging for LMS upload
What SCORM does: Wraps the HTML files in a package the LMS understands. The slide decks already contain SCORM 1.2 code — they will report "completed" to your LMS when a learner reaches the final slide.

Step 1 — Create the manifest file

Create a plain text file called imsmanifest.xml and place it in the root of the project folder (same level as index.html). Paste in this content:

<?xml version="1.0" encoding="UTF-8"?> <manifest identifier="acp2025-photoshop" version="1.2" xmlns="http://www.imsproject.org/xsd/imscp_rootv1p1p2"> <metadata> <schema>ADL SCORM</schema> <schemaversion>1.2</schemaversion> </metadata> <organizations default="org1"> <organization identifier="org1"> <title>ACP 2025 Photoshop</title> <item identifier="item1" identifierref="res1"> <title>Photoshop Simulator Slides</title> </item> </organization> </organizations> <resources> <resource identifier="res1" type="webcontent" adlcp:scormtype="sco" href="photoshop/slides/ps-simulator-slides.html"> <file href="photoshop/slides/ps-simulator-slides.html"/> </resource> </resources> </manifest>

To package the Illustrator or InDesign slides instead, change the highlighted href values to illustrator/slides/ai-simulator-slides.html or indesign/slides/id-simulator-slides.html.

Step 2 — Zip the project

  1. 1
    Select ALL files and folders inside the project folder. Right-click → "Compress" (Mac) or "Send to → Compressed (zipped) folder" (Windows).
    Critical: The imsmanifest.xml must be at the root of the zip — not inside a subfolder. If you compress the project folder itself, the manifest will be nested one level too deep and the LMS will reject it.
  2. 2
    Name the zip file something descriptive: acp-photoshop-slides.zip.

Step 3 — Upload to your LMS

The exact steps vary by platform, but the general process is:

Moodle
Course → Add activity → SCORM package → Upload the zip → Save. Set "Display package" to "New window".
Canvas
Modules → + → External Tool or use the "Upload" option in the LTI tool. Canvas uses its own SCORM player — check with your admin if SCORM is enabled.
Blackboard
Content → Build Content → SCORM Content Package → Upload the zip → Submit.

What learners see and how to use the system

Learner guide First-time walkthrough for learners
  1. 1
    Open the home page (index.html). A welcome popup explains the system — read it, then click "Get started".
    The popup only appears once. Your browser remembers you dismissed it.
  2. 2
    Start with the Design Theory module (the blue card near the top). This covers Domain 1 for all three tools — copyright, colour modes, print concepts, design principles.
    40 multiple-choice questions. Filter by topic. Instant answer feedback.
  3. 3
    Choose a tool: Photoshop, Illustrator, or InDesign. Click its card to see all domains.
  4. 4
    Inside each domain, click "Start Domain X" to open the interactive simulator.
    Simulators have two modes you can switch between at the top of each task:
    📚 Guided mode — step-by-step instructions with 3-level hints. Use this when learning.
    🎯 Independent mode — no instructions, simulates exam conditions. Use this for practice.
  5. 5
    In Guided mode: read the instruction, click the correct element in the simulated Adobe interface, and receive instant feedback. Click Hint 1 → Hint 2 → Hint 3 for progressively more help.
  6. 6
    After completing tasks, check the task briefs (found in each tool's "Files" section) for detailed answer keys, exact expected values, and a portfolio rubric.
  7. 7
    Before the real exam, watch the slide deck for each tool (also in the Files section). 20–30 slides, keyboard-navigable, with embedded MC knowledge checks and a final score.
  8. 8
    Progress dots appear on the home page after each completed task — green = passed (≥70%), amber = started, grey = not attempted.

Troubleshooting

Common issues Things that go wrong and how to fix them
The page looks unstyled — just plain text on a white background.
The browser cannot find assets/css/global.css. This happens when you open a file from inside a subfolder instead of from the root. Close everything and open index.html from the root of the project folder. Never open simulator files directly — always navigate to them from the home page.
Clicking a button does nothing in Guided mode.
The simulator expects a specific action. Read the step instruction carefully — it will tell you exactly which panel or button to click. Use the Hint buttons if you're stuck. If the entire page is unresponsive, try refreshing (F5) and reload the task from the sidebar.
My progress disappeared.
Progress is stored in the browser's localStorage. It can be lost if you: (a) clear browser cookies/cache, (b) use a different browser, (c) use a different computer, or (d) use a private/incognito window. For persistent cross-device progress, the system would need a backend database — that is out of scope for this v1 system.
The GitHub Pages URL shows a 404 error.
Wait 3–5 minutes after enabling Pages — it takes time to deploy. Also check that index.html is in the root of the repository (not inside a folder called "acp-2025-learning-system/"). If you uploaded the folder rather than its contents, the path will be wrong.
Fonts look wrong / different from the screenshots.
The system loads Google Fonts (DM Sans) from the internet. If the device has no internet access, the browser will fall back to a system sans-serif font. Functionality is not affected — only the visual appearance changes slightly.
The LMS rejects my SCORM zip upload.
The most common cause is that imsmanifest.xml is inside a subfolder rather than at the zip root. Unzip, check the structure (manifest should be at the same level as index.html), then re-zip by selecting all contents of the folder rather than the folder itself.
I want to add or edit content — where do I start?
Read EXPERT-PROMPT.md first — it documents the full architecture, task schema, and step-by-step instructions for extending the system. Each simulator is a self-contained HTML file with all its tasks defined in a const TASKS = {...} object near the bottom of the file.
